
# UI Wireframes & Component Specifications

## Overview
This document outlines the core user interface components and layouts for Agni Platform, with Agni Studio AI as the first product implementation.

## Core Pages & Views

### 1. Workspace Dashboard
**Purpose**: Central hub showing overview of all projects, upcoming deadlines, and quick actions

**Layout**:
- Sidebar: Navigation (Dashboard, Projects, Ideas, Content, Analytics, Settings)
- Top bar: Workspace name, search, notifications, user menu
- Main grid: 
  - Left column (2/3): Recent projects cards with progress indicators
  - Right column (1/3): 
    * Upcoming deadlines (calendar view)
    * Quick action buttons (+ New Idea, + New Project, + Upload Asset)
    * Today's focus (AI-suggested priority task)
    * Analytics snapshot (views today, engagement trend)

**Components**:
- ProjectCard: Shows project name, progress bar, last updated, upcoming deadline
- StatTile: Icon + number + label (e.g., ?? 3 Due This Week)
- ActionButton: Primary/secondary button with icon
- ActivityFeed: Recent actions across the workspace

### 2. Project View
**Purpose**: Detailed view of a specific content project or campaign

**Tabs**:
- Overview: Project description, goals, timeline, team members
- Pipeline: Kanban board of content ideas in various stages
- Calendar: Schedule view of planned/published content
- Assets: Media library for this project
- Analytics: Performance metrics for content in this project
- Settings: Project privacy, integrations, notifications

**Pipeline Kanban Board**:
- Columns: Ideas ? Research ? Writing ? Review ? Scheduled ? Published
- Cards show: Title, type (video/blog/etc.), assigned creator, due date, priority
- Drag-and-drop to move between columns
- Click card to open detailed editor

### 3. Idea Generation Interface
**Purpose**: AI-assisted brainstorming and idea development

**Layout**:
- Left sidebar (1/3):
  * Input area: Topic/seeds, audience, goals, content type
  * Filters: Trending topics, seasonal relevance, difficulty level
  * Inspiration feed: Related concepts, competitor gaps, audience questions
  
- Main panel (2/3):
  * Idea cards generated by AI Producer Agent
  * Each card shows: Title, brief description, AI confidence score, suggested format
  * Actions: Save to project, request research, combine with other ideas, discard
  * Infinite scroll or pagination
  
- Bottom bar: 
  * Selected ideas count
  * Bulk actions: Add to project, export, share for feedback

### 4. Research Assistant
**Purpose**: Information gathering, organization, and fact-checking

**Layout**:
- Top bar: Research query/topic, source type filters, depth selector
- Left pane (1/4):
  * Source list: Web results, documents, notes, bookmarks
  * Source type filtering (web, academic, news, social, uploads)
  - Credibility indicators for each source
  
- Center pane (1/2):
  - Note-taking area with rich text editing
  - Drag-and-drop from sources to notes
  - Auto-citation as you write
  - AI summarization of selected text
  
- Right pane (1/4):
  - AI Research Assistant chat panel
  - Suggested next questions
  - Fact-checking requests
  - Outline generation from notes
  - Bibliography/formatting help

### 5. Writing Environment
**Purpose**: Distraction-free content creation with AI writing assistance

**Layout**:
- Full-width editor with markdown/rich text options
- Left sidebar (collapsible): 
  * Outline navigator (auto-generated from headers)
  - Research references panel (drag citations in)
  - Version history
  
- Right sidebar (collapsible):
  - AI Writing Assistant:
    * Continue writing
    - Tone adjuster (more formal, more engaging, etc.)
    - Length changer (make shorter/longer)
    - Grammar & style improvement
    - Audience adaptation (simplify, expert-level, etc.)
  
- Bottom toolbar:
  - Word count, reading time, readability score
  - SEO suggestions (keyword usage, heading structure)
  - Tone analysis (confident, friendly, authoritative, etc.)
  - Plagiarism check button
  - Export options (copy, download, send to review)

### 6. Publishing Calendar
**Purpose**: Visual scheduling and cross-platform content distribution

**Views**:
- Month view: Traditional calendar with color-coded content types
- Week view: Horizontal timeline showing publishing slots
- List view: Agenda-style list with filtering options
- Platform view: Split view showing what's scheduled for each platform

**Features**:
- Drag-and-drop rescheduling
- Conflict detection (too many posts too close)
- Optimal time suggestions (based on historical engagement)
- Platform-specific preview (how it will look when published)
- Publishing queue status (ready, scheduled, publishing, failed, published)
- Bulk operations (reschedule multiple items, change platform)

### 7. Analytics Dashboard
**Purpose**: Performance insights and optimization recommendations

**Layout**:
- Top row: Key metrics summary (total views, engagement rate, growth, top content)
- Second row: Time series charts (views over time, engagement trend, follower growth)
- Third row: 
  * Content performance table (sortable by any metric)
  * Traffic sources pie chart
  * Audience demographics (age, location, interests)
- Bottom row:
  * AI-generated insights and recommendations
  * A/B test results
  * Competitive benchmarking (when available)

**Widgets**:
- MetricCard: Current value, change from previous period, sparkline
- ChartWrapper: Consistent styling for all charts with export options
- InsightCard: AI-generated recommendation with confidence level and expected impact
- DataTable: Sortable, filterable, paginated results with export

## Component Library

### Atoms (Basic Building Blocks)
- **Typography**: Heading levels, body text, caption, label, code
- **Buttons**: Primary, secondary, tertiary, icon-only, loading states
- **Inputs**: Text, textarea, select, checkbox, radio, toggle, date picker
- **Labels & Badges**: Status indicators, category tags, priority levels
- **Icons**: Standard set with consistent styling
- **Spacers & Dividers**: Horizontal rules, vertical spacing, flexible spaces

### Molecules (Groups of Atoms)
- **SearchBar**: Input + button + optional filters
- **FilterPanel**: Collapsible section with various input types
- **DataRow**: Clickable row for tables with action menu
- **ModalHeader**: Title + close button + optional actions
- **FormField**: Label + input + help text + validation message
- **NavItem**: Link/icon with active/inactive states
- **Avatar**: Image or initials with status indicator
- **LoadingSpinner**: Various sizes and styles

### Organisms (Complex UI Components)
- **ProjectCard**: As described in dashboard
- **StatTile**: Metric display with trend indicator
- **ContentPipelineBoard**: Drag-and-drop kanban with swimlanes
- **IdeaCard**: Rich preview of generated content idea
- **ResearchSource**: Display for web articles, documents, etc.
- **WritingToolbar**: Formatting options + AI assistant toggle
- **PublishingScheduler**: Calendar view with content cards
- **AnalyticsChartWrapper**: Consistent chart styling with loading/error states
- **AIChatInterface**: Message input, response display, suggestion chips
- **FileUploader**: Drag-and-drop area with progress indicators
- **MediaPreviewCard**: Container with header, body, footer sections
- **TabbedInterface**: Swipeable/clickable tab groups

### Templates (Page Layouts)
- **DashboardTemplate**: Sidebar + header + main content area
- **FormTemplate**: Two-column layout with main form and sidebar helpers
- **ListTemplate**: Filter toolbar + sortable list + pagination
- **DetailViewTemplate**: Header with actions + tabbed content sections
- **EditorTemplate**: Full-screen editor with collapsible sidebars
- **WizardTemplate**: Step-by-step progress with navigation

## Design System Specifications

### Color Palette
- **Primary**: #6366F1 (Indigo-500) - For main actions and branding
- **Secondary**: #10B981 (Emerald-500) - For positive actions and growth metrics
- **Accent**: #F59E0B (Amber-500) - For warnings and attention items
- **Background**: 
  - #FFFFFF (White) - Main content background
  - #F9FAFB (Gray-50) - Secondary backgrounds and cards
  - #F3F4F6 (Gray-100) - Disabled states and subtle separators
- **Text**:
  - #111827 (Gray-900) - Primary text
  - #6B7280 (Gray-500) - Secondary text
  - #9CA3AF (Gray-400) - Placeholder text and hints
- **Status**:
  - #EF4444 (Red-500) - Errors and critical issues
  - #FBBF24 (Amber-400) - Warnings and pending items
  - #10B981 (Emerald-500) - Success and completed items
  - #6366F1 (Indigo-500) - Info and neutral states

### Typography
- **Font Family**: 
  - Primary: 'Inter' or system UI font
  - Monospace: 'JetBrains Mono' or 'Fira Code' for code displays
- **Hierarchy**:
  - H1: 2.25rem (36px) - Page titles
  - H2: 1.875rem (30px) - Section headers
  - H3: 1.5rem (24px) - Card titles
  - H4: 1.25rem (20px) - Subsection headers
  - Body: 1rem (16px) - Main text
  - Small: 0.875rem (14px) - Captions and helper text
  - XS: 0.75rem (12px) - Fine print and timestamps
- **Weights**: 
  - Regular: 400
  - Medium: 500
  - Semi-bold: 600
  - Bold: 700
  - Extra-bold: 800

### Spacing & Layout
- **Base Unit**: 4px (all spacing multiples of this)
- **Container Padding**: 
  - Desktop: 2rem (32px) horizontal
  - Tablet: 1.5rem (24px) horizontal
  - Mobile: 1rem (16px) horizontal
- **Vertical Rhythm**: 
  - Section spacing: 2rem (32px)
  - Element spacing: 1rem (16px) 
  - Compact spacing: 0.5rem (8px)
- **Border Radius**:
  - Small: 0.25rem (4px) - Inputs, buttons
  - Medium: 0.5rem (8px) - Cards, modals
  - Large: 1rem (16px) - Panels, dialogs
  - Circular: 50% - Avatars, circular buttons

### Elevation & Shadows
- **Level 0**: No shadow (inline elements)
- **Level 1**: 0px 1px 3px rgba(0,0,0,0.1), 0px 1px 2px rgba(0,0,0,0.06) - Cards, buttons
- **Level 2**: 0px 4px 6px -1px rgba(0,0,0,0.1), 0px 2px 4px -1px rgba(0,0,0,0.06) - Floating elements
- **Level 3**: 0px 10px 15px -3px rgba(0,0,0,0.1), 0px 4px 6px -2px rgba(0,0,0,0.05) - Modals, drawers
- **Level 4**: 0px 20px 25px -5px rgba(0,0,0,0.1), 0px 10px 10px -5px rgba(0,0,0,0.04) - Full-screen overlays

### Interactive States
- **Hover**: +2% brightness increase for backgrounds, subtle scale transform
- **Focus**: 2px solid outline using primary color, offset by 2px
- **Pressed**: -2% brightness decrease, slight scale down
- **Disabled**: Saturation reduced to 20%, opacity 50%
- **Selected**: Background tint with primary color at 10% opacity

### Motion & Animation
- **Duration Standards**:
  - Fast: 100ms (button presses, toggle switches)
  - Normal: 200ms (page transitions, modal fades)
  - Slow: 300ms (drawer slides, complex transformations)
  - Below: 400ms (entrance/exit of major views)
- **Easing Functions**:
  - Standard: cubic-bezier(0.4, 0, 0.2, 1) (Material ease-out)
  - Acceleration: cubic-bezier(0.4, 0, 1, 1)
  - Deceleration: cubic-bezier(0, 0, 0.2, 1)
  - Sharp: cubic-bezier(0.4, 0, 0.6, 1)
- **Page Transitions**: Fade + slight slide (100ms fade, 200ms slide)
- **Micro-interactions**: Button presses, form validations, hover effects

## Responsive Breakpoints
- **Mobile**: < 640px

- **Desktop**: = 1024px

## Platform-Specific Considerations

### Web App (Primary)
- Full feature set as described above
- Keyboard navigation and shortcuts
- Export/print capabilities
- Advanced filtering and sorting

### Mobile App (Future)
- Priority views: Dashboard, Calendar, Inbox (notifications/comments)
- Simplified creation flows for quick capture
- Camera/gallery integration for asset collection
- Offline capability for viewing and light editing
- Push notifications for publishing reminders and team mentions

### Accessibility (WCAG 2.1 AA)
- Color contrast ratios = 4.5:1 for normal text, = 3:1 for large text
- All functionality available via keyboard
- ARIA labels for interactive elements
- Screen reader friendly structure
- Resizable text up to 200% without loss of functionality
- Focus visible indicators
- Error identification and suggestions
- Consistent navigation and identification

## Implementation Guidelines

### Frontend (React/Next.js)
- **Component Organization**:
  `
  components/
    atoms/
    molecules/
    organisms/
    templates/
    pages/
  `
- **Styling**: Tailwind CSS with custom configuration for design tokens
- **State Management**: React Query for server state, Context/Zustand for client state
- **Type Safety**: Extensive TypeScript interfaces for all props and state
- **Performance**: Lazy loading, code splitting, image optimization
- **Testing**: Jest + React Testing Library for unit/react testing, Cypress for e2e

### Backend (Laravel/API)
- **Response Format**: Consistent JSON structure with data/meta/errors
- **Validation**: Form Requests for all incoming data
- **Authorization**: Policies and gates for resource access
- **Resources**: API Resources for consistent output formatting
- **Events**: Broadcasting for real-time updates (WebSocket/Echo)
- **Queues**: For heavy processing (AI generation, media processing)
- **Testing**: PHPUnit for feature and unit tests
- **Documentation**: OpenAPI/Swagger generation from annotations

## Internationalization & Localization
- **UI Strings**: Externalized for translation
- **Date/Time**: Locale-aware formatting
- **Number Formats**: Localized decimal and grouping separators
- **RTL Support**: Prepared layout mirroring for right-to-left languages
- **Regional Settings**: Time zone, currency, measurement units per user/workspace

## Feedback Mechanisms
- **In-app Surveys**: Contextual feedback after feature usage
- **Analytics Integration**: Feature usage tracking for prioritization
- **Error Reporting**: Automatic crash reporting with user consent
- **Feature Requests**: Built-in idea board for user suggestions
- **Beta Testing**: Opt-in channels for upcoming features

## Success Metrics for UI/UX
- **Task Completion Rate**: = 80% of users can complete core flows without assistance
- **Time to First Value**: < 5 minutes to create and schedule first content piece
- **Error Rate**: < 5% of form submissions have validation errors
- **Feature Discovery**: = 60% of users find advanced features without guidance
- **Satisfaction Score**: = 4.0/5.0 in post-session surveys
- **Retention**: = 40% week-over-week return rate for core features
- **Accessibility Score**: = 90% on automated axe-core testing
